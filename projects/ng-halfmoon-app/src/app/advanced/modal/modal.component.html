Modals

Basic Usage

Basically you need to create a CustomModalComponent that you will paste into a modal-container.
Next you need to inject the ModalService to create this modal on the fly.
Therefore you have to call the createModal-Method on the service.
You'll paste your custom component into the method like so: modalService.createModal(CustomModalComponent).

Dismiss

By default every modal is dismissable by clicking outside of the modal.
You can alternatively add a flag to the createModal-Method to add the default dismiss-button.
You need to add it as follows createModal(Cmp, {{'{defaultDismiss: true}'}})

Inject custom Data

When you are creating a modal you can inject custom data into your modal-component.
You will need to add the data-property inside of the modal-options. The property is variable so you can send a simple value or a complex object.
You need to add it as follows createModal(Cmp, {{'{data: {name: "John"}'}}'}}) Inside of the ModalComponent you can then inject the data via the MODAL_DATA-Token as follows constructor(@Inject(MODAL_DATA) private data: Record<string, unknown>)

Return Data on close

Sometimes modals are created to input some sort of data or to ask the user if he is leaving the site on purpose.
Therefore you will need to return the outcome of the action. You can simply save the ModalRef of createModal into a variable to use it later.
Then you will need to subscribe onto the afterClosed-Event to apply your logic once the modal is closed.
Via the afterClosed you can get data that you are passing from your modal. Inside the modal you can call the modalRef.close with a parameter to send your data.
